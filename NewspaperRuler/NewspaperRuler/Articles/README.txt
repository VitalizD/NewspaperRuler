При написании игровых статей следует придерживаться следующих правил:

1. Формат файла должен быть .txtпорядковый
2. Файл обязательно положить в папку соответствующего уровня, на котором она должна встретиться.
3. Все теги пишутся перед основным текстом статьи.
4. Формат написания тега: 
[имя тега]: [значение]
5. Перед основным текстом обязателен тег "Content:".
После этого тега делается перенос на следующую строку и пишется основной текст.
6. Формат названия файла:
Article[номер уровня]_[порядковый номер статьи]


ОПИСАНИЕ ТЕГОВ

Loyality: 
* Прибавка к лояльности в случае одобрения статьи. 
* Если тег не равен 0, то в случае отклонения статьи будет вычтена 1 ед. лояльности.
* Принимает значения целых чисел, больше или равных 0.
* Значение по умолчанию: 0.

Reprimand:
* Прибавка к очкам выговора в случае одобрения статьи.
* Если в результате уровня количество очков выговора равно 3 или выше, задание от Министерства считается проваленным.
* Принимает значения целых чисел, больше или равных 0.
* Значение по умолчанию: 0.

Order:
* Порядок очерёдности статьи. Показывает, что событием под данным номером точно окажется эта статья.
* Принимает значения целых чисел, больше или равных -1.
* Значение -1 означает случайную установку очерёдности.
* Значение по умолчанию: -1.

Mistake:
* Тип ошибки, содержащейся в данной статье.
* Алгоритм сгенерирует замечание в случае одоберения статьи с этим тегом.
* Принимает строковое значение, которое впоследствии преобразуется в тип перечисления.
* Значение по умолчанию: None

Flag:
* Устанавливает метку на событие в случае одобрения статьи. 
* Метки используются для формирования последующих сюжетных событий.
* Принимает строковое значение.

Title:
* Название статьи.
* Принимает строковое значение.
* Значение по умолчанию: "".

Genre:
* Предметная область статьи.
* Принимает строковое значение.
* Значение по умолчанию: "".

Content:
* Обязательный тег, после которого делается перенос строки и пишется основной текст.
* После двоеточия на той же строке не пишется НИЧЕГО.
